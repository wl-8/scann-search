# 前端使用说明（Frontend 使用说明）

本文档位于项目 `frontend` 目录下，帮助开发者快速在本地启动、测试和与后端联调。

## 先决条件

- Node.js 18+（建议）
- npm 9+
- 可选：已启动的后端服务（若要联调真实后端）

## 快速开始

1. 进入前端目录：

```bash
cd frontend
```

2. 安装依赖：

```bash
npm install
```

3. 开发模式运行（启用 mock 或与后端联调）：

```bash
npm run dev
```

- 默认开发服务器会启用内置 mock 中间件（参见 `vite.config.ts`），便于在没有后端时演示功能。
- 若要对接真实后端，请在 `frontend` 根目录创建 `.env`（或修改 `.env`）并设置：

```env
VITE_API_BASE=http://localhost:8000/api
```

然后重启开发服务器。

## 完整启动项目

如果你要从零把整个项目跑起来，建议按下面顺序执行。

### 1. 启动后端

进入后端目录并安装依赖：

```bash
cd ../backend
pip install -r requirements.txt
```

启动 FastAPI 服务：

```bash
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

启动成功后，可以先确认接口可访问：

```bash
curl http://localhost:8000/api
```

如果你的后端不是 8000 端口，请把实际地址记下来，后面要写进前端 `.env`。

### 2. 配置前端环境变量

回到 `frontend` 目录，检查或创建 `.env` 文件：

```bash
cd ../frontend
```

建议内容如下：

```env
VITE_API_BASE=http://localhost:8000/api
```

如果后端端口不同，请改成你的实际地址，例如 `http://localhost:8001/api`。

### 3. 启动前端

安装依赖（只需一次）：

```bash
npm install
```

启动开发服务器：

```bash
npm run dev
```

看到 `http://localhost:5173/` 之类的地址后，用浏览器打开即可。

### 4. 验证联调是否成功

建议按这个顺序检查：

1. 打开 `/login`，确认能登录并跳转到仪表盘。
2. 打开 `/datasets`，确认能看到数据集列表，注册或删除数据集时没有报错。
3. 打开 `/indexes`，确认能加载算法列表和索引列表。
4. 打开 `/search`，确认“后端资源加载失败”不再出现，并且可以选择数据集和索引。
5. 打开 `/visualize`，确认能加载散点图并点击点位查看相似细胞。

### 5. 测试

如果你要检查前端单测：

```bash
npm run test
```

如果终端进入了 watch 模式，按 `q` 退出。

### 6. 停止服务

- 停止前端：在运行 `npm run dev` 的终端按 `Ctrl + C`
- 停止后端：在运行 `uvicorn` 的终端按 `Ctrl + C`

## 与后端对接要点

- 后端基础路径由环境变量 `VITE_API_BASE` 控制（例如 `http://localhost:8000/api`）。
- 请求会自动携带 `Authorization: Bearer <token>`（从 `localStorage.token` 读取）。
- 关键接口（参见 `src/api`）：
  - 认证：`POST /auth/login`、`POST /auth/logout`、`GET /auth/me`（封装在 `src/api/auth.ts`）
  - 数据集：`GET /datasets`、`POST /datasets/register`、`DELETE /datasets/{id}`（封装在 `src/api/datasets.ts` / `src/api/search.ts`）
  - 索引：`GET /index`、`POST /index/build`、`DELETE /index/{id}`、`GET /index/algorithms`（封装在 `src/api/indexes.ts`）
  - 检索：`POST /search/by-cell`、`POST /search/by-vector`、`POST /search/batch`、`GET /visualize/{dataset_id}/embedding`（封装在 `src/api/search.ts`）

- 如果后端返回 401，前端会自动跳转到 `/login`（见 `src/api/request.ts` 的响应拦截器）。

## 常见操作

- 登录：访问 `/login` 页面，任意用户名密码可在本地演示登录（如需真实认证，确保后端实现 `/auth/login`）。
- 加载数据集与索引：在 `/search` 或 `/datasets` 页面点击刷新按钮，会调用 `GET /datasets` 与 `GET /index`。
- 注册数据集：在数据集页面使用上传或填写服务器路径后点击注册，触发 `POST /datasets/register`。
- 构建索引：在 `/indexes` 页面选择数据集和算法，点击构建（触发 `POST /index/build`）。
- 可视化：在 `/visualize` 页面点击“刷新图谱”会调用 `GET /visualize/{dataset_id}/embedding` 并渲染 UMAP。

## 调试与排错

- 若页面空白或组件报错：先确认是否在 `frontend` 目录运行了 `npm run dev`。
- 若无法联通后端：检查 `.env` 中的 `VITE_API_BASE` 是否正确，后端是否运行并监听对应端口；如果浏览器提示“后端资源加载失败”，通常说明 FastAPI 没有启动或地址写错了。
- 若接口返回 403/401：清除浏览器 `localStorage.token` 并重新登录；检查后端鉴权配置。
- 查看浏览器控制台与网络请求（Network）以获取失败请求的详细信息。

## 文件与约定

- `src/api/`：后端接口封装，请优先修改这里以统一调用方式。
- `src/views/`：页面入口文件（`/search`、`/visualize`、`/datasets`、`/indexes`、`/login`）。
- `src/stores/auth.ts`：登录态管理（token 存储与 `fetchCurrentUser`）。
- `.env.example`：示例环境变量（`VITE_API_BASE`）。
