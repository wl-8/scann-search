# 前后端对接清单

这份清单用于前端与后端联调时快速对照页面、接口、字段与待确认项。当前状态以本仓库现有实现为准。

## 1. 总体约定

- 前端统一使用 `VITE_API_BASE` 作为后端基础地址，当前示例为 `http://localhost:8000/api`。
- 请求默认携带 `Authorization: Bearer <token>`。
- 登录态由 `localStorage.token` 维护，启动时会调用 `/auth/me` 恢复用户信息。
- 受保护页面统一使用路由守卫，未登录跳转 `/login`，管理员页面额外检查 `role === admin`。

## 2. 页面与接口映射

| 页面 | 前端文件 | 后端接口 | 说明 |
| --- | --- | --- | --- |
| 登录页 | `src/views/auth/LoginView.vue` | `POST /auth/login`、`POST /auth/logout`、`GET /auth/me` | 登录后保存 token，进入仪表盘 |
| 仪表盘 | `src/views/dashboard/DashboardView.vue` | `GET /auth/me` | 主要用于登录后入口页 |
| 数据集管理 | `src/views/datasets/DatasetListView.vue` | `GET /datasets`、`POST /datasets/register`、`DELETE /datasets/{id}` | 注册后刷新列表 |
| 搜索页 | `src/views/search/SearchView.vue` | `GET /datasets`、`GET /index`、`POST /search/by-cell`、`POST /search/by-vector` | 支持按索引检索 |
| 多搜索页 | `src/views/search/MultiSearchView.vue` | `POST /search/batch` | 当前前端将多查询汇聚为 batch 语义 |
| 可视化页 | `src/views/visualize/VisualizeView.vue` | `GET /datasets`、`GET /visualize/{dataset_id}/embedding`、`POST /search/by-cell` | 点击点后联动检索 |
| 索引页 | `src/views/index/*` | `GET /index`、`POST /index/build`、`DELETE /index/{id}` | 当前路由仍待补全 |

## 3. 认证对接

### 3.1 登录

- 前端请求：`POST /auth/login`
- 请求体：

```json
{ "username": "demo", "password": "123456" }
```

- 后端返回：

```json
{ "access_token": "...", "token_type": "bearer" }
```

- 前端处理：
  - 保存 `access_token`
  - 请求 `/auth/me` 获取当前用户
  - 跳转 `/dashboard`

### 3.2 当前用户

- 前端请求：`GET /auth/me`
- 返回字段：

```json
{
  "id": 1,
  "username": "admin",
  "email": "admin@scann.local",
  "role": "admin",
  "review_status": "approved",
  "account_status": "active"
}
```

### 3.3 退出登录

- 前端请求：`POST /auth/logout`
- 后端行为：无状态注销，客户端清理 token 即可

## 4. 数据集对接

### 4.1 数据集列表

- 前端请求：`GET /datasets`
- 用途：数据集页、搜索页、可视化页加载下拉列表

### 4.2 注册数据集

- 前端请求：`POST /datasets/register`
- 请求体：

```json
{
  "name": "PBMC-3k",
  "source_path": "D:/data/pbmc.h5ad",
  "description": "",
  "embedding_key": "X_pca"
}
```

- 返回字段：`DatasetResponse`
  - `id`
  - `name`
  - `description`
  - `source_path`
  - `n_cells`
  - `n_genes`
  - `vector_dim`
  - `embedding_key`
  - `status`
  - `error_msg`
  - `created_at`

### 4.3 删除数据集

- 前端请求：`DELETE /datasets/{id}`
- 前端处理：删除后刷新列表

### 4.4 数据集状态

- `processing`：后端正在解析或落盘
- `ready`：可用于检索/索引/可视化
- `error`：注册失败，前端需要展示 `error_msg`

## 5. 索引对接

### 5.1 索引列表

- 前端请求：`GET /index`
- 可选查询：`dataset_id`
- 返回字段：`IndexResponse[]`

### 5.2 构建索引

- 前端请求：`POST /index/build`
- 请求体：

```json
{
  "dataset_id": 1,
  "algorithm": "hnsw",
  "params": { "M": 16, "ef_construction": 200, "ef_search": 50 }
}
```

- 返回字段：`IndexResponse`

### 5.3 删除索引

- 前端请求：`DELETE /index/{id}`

### 5.4 算法列表

- 前端请求：`GET /index/algorithms`
- 当前后端支持：`flat`、`hnsw`、`ivf`

## 6. 检索对接

### 6.1 按 cell 检索

- 前端请求：`POST /search/by-cell`
- 请求体核心字段：

```json
{
  "index_id": 1,
  "cell_id": "cell_123",
  "k": 10,
  "oversample": 10,
  "metric": "l2",
  "filters": {
    "equals": { "cell_type": ["T-cell"] },
    "gte": {},
    "lte": {}
  }
}
```

### 6.2 按向量检索

- 前端请求：`POST /search/by-vector`
- 请求体与上面类似，`cell_id` 换成 `vector: number[]`

### 6.3 批量检索

- 前端请求：`POST /search/batch`
- 前端用于多查询聚合结果展示

### 6.4 条件检索

- 当前前端封装的 `conditionalSearch` 实际复用 `/search/by-cell` 或 `/search/by-vector`
- 如果后端后续补出专用接口，前端只需要改 `src/api/search.ts`

### 6.5 搜索返回字段

- 顶层常见字段：`index_id`、`dataset_id`、`algorithm`、`metric`、`k`、`n_returned`、`latency_ms`、`filter_applied`、`hits`
- `hits[]` 常见字段：
  - `rank`
  - `cell_id`
  - `row_index`
  - `distance`
  - `obs`

## 7. 可视化对接

### 7.1 拉取 embedding

- 前端请求：`GET /visualize/{dataset_id}/embedding`
- 查询参数：
  - `mode=2d|3d`
  - `color_by=cell_type|dataset|disease|...`
  - `max_points=5000`

- 返回字段：
  - `points[]`
  - `n_returned`
  - `color_options`
  - `embedding_key`
  - `mode`
  - `dataset_id`

### 7.2 点击点联动检索

- 前端行为：点击散点后调用 `POST /search/by-cell`
- 用途：右侧显示相似细胞结果

## 8. 当前已完成与待确认

### 已完成

- 登录、登出、当前用户获取
- token 存储与路由守卫
- 数据集列表、注册、删除
- 索引列表、构建、删除
- 搜索页与可视化页的后端联调
- WebGL 可视化与点击联动

### 待确认

- 浏览器直传 multipart 上传是否需要后端新增 `POST /datasets/upload`
- `/search/conditional`、`/search/multi-dataset` 是否要由后端提供专用路由，还是继续沿用现有组合封装
- 索引管理页面是否要补齐路由与 UI
- 401 / 403 的统一错误页是否要单独做成页面

## 9. 给联调同学的最小检查顺序

1. 登录成功后确认 `/auth/me` 能返回角色与账号状态。
2. 打开数据集页，确认 `GET /datasets` 能返回列表，注册成功后列表刷新。
3. 打开搜索页，确认选中数据集与索引后能正常返回 `hits`。
4. 打开可视化页，确认 `GET /visualize/{dataset_id}/embedding` 能返回 `points`。
5. 点击散点，确认 `/search/by-cell` 的结果能在右侧展示。
