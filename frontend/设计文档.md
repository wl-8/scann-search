

# 🧬 面向单细胞数据的 ANN 检索系统 - 前端设计与需求文档

## 一、 开发规范与基础要求


**应用形式**：系统必须以 Web 应用的形式呈现 。



**技术栈选择**：前端技术栈无强制限制，若无特定 Web 开发框架经验，建议配合 Flask 框架进行开发 。



**版本控制**：必须使用 Git 进行代码的版本管理 ，并提交至 GitHub，要求包含完整的提交记录以体现项目的完成过程 。



---

## 二、 核心功能模块划分与 UI 设计

### 1. 用户信息模块


**账号管理**：提供用户通过账号和密码进行注册与登录的页面 。



**权限划分**：区分普通用户与管理员。普通用户主要访问可视化系统，管理员需要有专属的后台管理界面，用于进行用户管理等操作 。



### 2. 数据与索引管理模块 (后台/控制台界面)


**数据集管理**：提供可视化界面，支持管理员或用户进行数据集的增加、删除等动态管理操作 。



**状态反馈**：前端界面需展示单细胞数据的读取、格式校验与基础预处理状态 ，并动态展示底层索引的构建、保存和加载状态 。



### 3. 查询检索模块 (核心交互界面)


**基础查询输入**：提供核心输入组件，支持用户输入“查询细胞编号”或“查询向量” 。


**参数设置面板**：允许用户自定义检索参数，必须包含以下设置项：
索引类型 。


距离度量方式 。


返回结果数量（Top-K） 。





**条件过滤（结项重点）**：增加条件检索的 UI 组件，支持用户根据特定条件要求（如限定某具体细胞类型）对返回的 Top-K 相似细胞结果进行过滤 。



### 4. 可视化展示模块


**结果列表/表格**：以直观的形式展示检索返回的 Top-K 个最相近细胞的结果及其对应的细胞元信息 。



**性能仪表盘**：展示具体的查询耗时（响应时间）等实验评估性能分析指标 。



**降维可视化图表**：提供至少一种可视化结果展示形式（例如基于 PCA 或 UMAP 降维后的散点图） 。



**图表交互操作（鼓励项）**：鼓励在前端实现简单的交互方式，例如允许用户直接在可视化结果图表上点击，从而发起针对该细胞的相似性查询 。

## 三、 进阶与加分项 (前端界面支持)

基础要求以外的其他功能可视完成情况加分 。若团队计划争取加分项，前端需预留或开发以下功能组件：


**跨数据集检索选项**：在检索面板中增加选项卡或多选组件，支持用户针对多个单细胞数据集联合建立的索引进行跨数据集搜索 。



**AI 辅助问答界面**：开发类似自然语言对话框的 UI，支持用户使用自然语言查询细胞，并展示 RAG 结合单细胞数据库生成的辅助细胞分析结果 。

## 四、 提交规范与文档配合要求

### 1. 中期提交阶段


**展示形式**：需要准备 5-8 分钟的现场演示或录制视频，重点在于展示已完成的功能界面与操作流程 。



**前端展示重点**：确保单细胞数据读取、单细胞数据向量化表示、ANN 索引构建和相似细胞检索的流程能在 Web 界面上形成初步的闭环演示 。
